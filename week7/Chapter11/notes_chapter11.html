  • functions are first-class objects, which means they can be passed around in the same way as every other value. They can have their own properties and methods, as well as accepting other functions as parameters and being returned by other functions.

  • Closure - is a reference to a variable that was created inside the scope of another function, but is then kept alive and used in another part of the program
  • Generators - are special functions used to produce iterators that maintain the state of a value. To define a generator function, an asterisk symbol (*) is placed after the function declaration
  • Functional Programing:
    1. A key aspect of functional programming is its use of pure functions. A pure function is a function that adheres to the following rules:
    2. 1) The return value of a pure function should only depend on the values provided as arguments. It doesn't rely on values from somewhere else in the program.
    3. 2) There are no side-effects. A pure function doesn't change any values or data elsewhere in the program. It only makes non-destructive data transformations and returns new values, rather than altering any of the underlying data.
    4. 3) Referential transparency. Given the same arguments, a pure function will always return the same result.
    5. In order to follow these rules, any pure function must have:
    6. At least one argument; otherwise the return value must depend on something other than the arguments of the function, breaking the first rule
    7. A return value; otherwise there’s no point in the function (unless it has changed something else in the program – in which case, it’s broken the 'no side-effects' rule).
  • Higher-Order Functions - functions that accept another function as an argument, or return another function as a result, or both
  • Currying - not all arguments have been supplied to the function, so it returns another function that retains the arguments already provided, and expects the remaining arguments that were omitted when the original function was called. A final result is only returned once all the expected arguments have eventually been provided.
  • Currying is the idea of creating a funciton that can be redefined if an agruemnt has not been supplied.

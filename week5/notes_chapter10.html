Chapter 10

Resources for checking code:

  • Use strict mode, js Lint, or ESLint
  • JS Lint or ESLint
  • check to see if a feature is supported in a browser use Modernizr or Can I Use?
  • Using the console is a very good way to debug code.
  • Use the debugger command to set breakpoints in your code
  • You can create your own error objects and use them to throw your own errors
  • Using the Errors you can do Exception Handling using try, catch and finally
  • You will want to create and run tests on your code to make sure it is working properly
  • Test-driven development:
    1. Write tests (that initialy fail)
    2. Write code to pass the tests
    3. Refactor the code
    4. Test refacterd code
    5. Write more test for new features

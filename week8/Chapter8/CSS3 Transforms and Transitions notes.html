Chapter 8: CSS3 Transforms and Transitions

  • The transform property gives you the ablity to translate, rotate, scale or skew and element on the page.
    1. Translation: allows you to move and object left and right up and down. It is written like so: translate(x,y) x from the left and y from the top. tanslateY() and translateX() can be used to only move along one axis
    2. Scaling: scale(x,y) increases the size of an element. When uing this only the element selected will grow elments around it will not change to fit the new size.
    3. Rotation: rotate() rotates an element around the point of origin by a specified angle value. The point from which is rotates is the elments center.
    4. Skew: skew(x,y) function specifies a skew along the x and y axes. The x specifies the skew on the x axis, and the y specifies the skew on the y axis. If the second parameter is omitted, the skew will only occur on the x axis
    5. transform-origin redefines from which point with in your element your transform happens.
    6. Transforms are all declared on one line and are applied in the order they are placed from right to left.
    7. Cross browser code for a transform
  • Transistions is the idea of telling a browers how you want a transform to change from one state to another
    1. rather than writing out an animation in JavaScript, you can simply switch a property value and the browser will do the transistion as specified.
    2. transistion-porperty: the defualt is all but if there is a specific change on an element you want to be transistioned than list it here seperated by commas. A full list can be found here. It is best to specify what you want to transistion as something in the future may get the ablity and mess up your site.
    3. transition-duration: specifies the time it takes to transfom. 200ms is considered the optimum time.
    4. transiton-timing-function: this lets you control if it speeds up or slows downa and how you want that to be.
    5. transition-delay: this gives the transition a delayed start.
    6. transitions have a shorthand and can be written in any order on the same line but if you have a delay you must include a deration.
  • Animations- allows you to control each stage of the animation using keyframes
    1. You first have to define the animantion using keyframes before applying it to an element.
    2. animation-name: a property that you use to speifiy which @keyframes you want.
    3. animantion-duration: a property to define the length of time for one iteraiton of the @keyframes
    4. animation-timing-function: a property to determine how the animation will progress over time.
    5. animation-iteration-count: a property to define how many times you want it do iterate throught @keyframes
    6. animation-direction: a property allows you to specify if an animaiton after one go through will go throgh backwards or not.
    7. animaiton-delay: a proprty to define if you want a delay before beging an animation.
    8. animation-fill-mode: allows you to define what the animaiton is to do during the delay.
    9. There is short hand where all of this can be written on one line only seperated by a space. Commas are used to sperate differnt animaations for one element.
